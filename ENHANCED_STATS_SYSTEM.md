# Улучшенная система отчетов о ежедневных играх

Система отчетов была значительно расширена для предоставления более детальной аналитики о пользователях и их участии в ежедневных играх.

## Новые возможности

### 1. Статистика за период (`/period_stats`)
Получает комплексную статистику за указанный период времени.

**Использование:**
- `/period_stats` - статистика за последние 7 дней (по умолчанию)
- `/period_stats 30` - статистика за последние 30 дней
- `/period_stats 2024-01-01 2024-01-31` - статистика за конкретный период

**Что показывает:**
- Общая статистика (всего игр, уникальных пользователей, ответов, правильных ответов, очков)
- Детальная статистика по каждому типу игры (слово, идиома, phrasal verb, квиз)
- Процент участия и точность для каждого типа игры
- Среднее время ответа
- Топ-10 самых активных пользователей за период

### 2. Статистика пользователя (`/user_stats`)
Получает детальную статистику конкретного пользователя.

**Использование:**
- `/user_stats [user_id]` - статистика за последние 30 дней
- `/user_stats [user_id] 7` - статистика за последние 7 дней
- `/user_stats [user_id] 60` - статистика за последние 60 дней

**Пример:** `/user_stats 123456789 30`

**Что показывает:**
- Общая статистика пользователя (всего игр, ответил, правильно, пропустил)
- Процент участия и точность
- Всего заработанных очков
- Среднее время ответа
- Детальная статистика по каждому типу игры
- Количество активных дней

### 3. Топ активных пользователей (`/top_users`)
Показывает самых активных пользователей за период.

**Использование:**
- `/top_users` - топ-10 за последние 30 дней
- `/top_users 20` - топ-20 за последние 30 дней
- `/top_users 15 7` - топ-15 за последние 7 дней

**Что показывает:**
- Рейтинг пользователей по очкам
- Количество игр и ответов
- Процент точности
- Количество активных дней

## Улучшения существующих отчетов

### Ежедневный отчет (`/word_stats`)
Теперь включает:
- Процент участия для каждого типа игры
- Среднее время ответа
- Улучшенное форматирование с более детальной информацией
- Ограничение списка участников (показывается до 20, остальные в сводке)

## Новые функции в API

### `getPeriodStats(startDate, endDate, gameType)`
Получает статистику за период с группировкой по:
- Типам игр
- Датам
- Пользователям

### `getUserDetailedStats(userId, startDate, endDate, days)`
Получает детальную статистику пользователя с разбивкой по:
- Типам игр
- Датам
- Индивидуальным играм

### `getTopActiveUsers(limit, days)`
Возвращает топ самых активных пользователей с метриками:
- Общее количество игр
- Процент участия
- Точность ответов
- Количество активных дней

### `comparePeriods(period1Start, period1End, period2Start, period2End)`
Сравнивает два периода и показывает изменения в процентах.

## Примеры использования

### Получить статистику за неделю
```
/period_stats 7
```

### Получить статистику конкретного пользователя
```
/user_stats 123456789 30
```

### Найти самых активных пользователей за месяц
```
/top_users 20 30
```

### Получить статистику за конкретный месяц
```
/period_stats 2024-01-01 2024-01-31
```

## Права доступа

Все новые команды доступны только администраторам бота (проверяется через `ADMIN_ID`).

## Технические детали

### Производительность
- Все запросы оптимизированы с использованием индексов базы данных
- Группировка данных выполняется в памяти для быстрого доступа
- Длинные сообщения автоматически разбиваются на части (максимум 4000 символов)

### Форматирование
- Все отчеты используют HTML-разметку для лучшей читаемости
- Эмодзи используются для визуального разделения секций
- Проценты и метрики округляются для удобства чтения
