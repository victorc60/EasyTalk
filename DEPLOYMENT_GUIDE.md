# Руководство по деплою

## Нужен ли Docker на MacBook?

**Короткий ответ: НЕТ, не нужен для деплоя.**

Docker на вашем MacBook **не требуется** для деплоя на облачную платформу (Railway, Fly.io, Render и т.д.), потому что:

1. ✅ Сборка Docker образа происходит **на стороне платформы**
2. ✅ Платформа сама использует Docker для сборки и запуска
3. ✅ Вам нужно только загрузить код (через Git или веб-интерфейс)

## Когда Docker на MacBook может быть полезен?

Docker локально нужен **только если** вы хотите:

- 🧪 **Локально протестировать** сборку перед деплоем
- 🐛 **Отладить проблемы** локально
- ✅ **Убедиться**, что Dockerfile работает корректно

Но это **опционально** - не обязательно для деплоя!

## Что делать, если деплой не работает?

### 1. Проверьте логи на платформе

Найдите раздел с логами запуска контейнера (не сборки). Ошибки обычно там.

**Что искать:**
- Ошибки подключения к базе данных
- Отсутствующие переменные окружения
- Ошибки при запуске `npm start`

### 2. Проверьте переменные окружения

Убедитесь, что на платформе установлены **все** необходимые переменные:

```
TELEGRAM_BOT_TOKEN=ваш_токен
OPENAI_API_KEY=ваш_ключ
ADMIN_ID=ваш_telegram_id
DATABASE_URL=строка_подключения
DB_HOST=хост_базы
DB_NAME=имя_базы
DB_USER=пользователь
MYSQLPASSWORD=пароль
```

### 3. Проверьте статус контейнера

Убедитесь, что контейнер:
- ✅ Успешно собрался
- ✅ Запустился (не падает сразу)
- ✅ Имеет доступ к интернету (для Telegram API)

## Локальное тестирование (опционально)

Если хотите протестировать локально:

### Установка Docker Desktop

1. Скачайте Docker Desktop для Mac: https://www.docker.com/products/docker-desktop
2. Установите и запустите приложение
3. Дождитесь, пока Docker запустится (иконка в меню должна быть зеленая)

### Тестирование сборки

```bash
# Запустите скрипт тестирования
./test-docker-build.sh

# Или вручную:
docker build -t easytalk-bot-test .
```

### Тестирование запуска

```bash
# Убедитесь, что .env файл существует
docker run --env-file .env easytalk-bot-test
```

## Частые проблемы при деплое

### Проблема: "Container exited with code 1"

**Причина:** Ошибка при запуске приложения

**Решение:**
- Проверьте логи контейнера
- Убедитесь, что все переменные окружения установлены
- Проверьте подключение к базе данных

### Проблема: "Cannot find module"

**Причина:** Отсутствуют файлы или зависимости

**Решение:**
- Убедитесь, что все файлы загружены в репозиторий
- Проверьте `.dockerignore` - не исключены ли важные файлы
- Убедитесь, что `package.json` содержит все зависимости

### Проблема: "Database connection failed"

**Причина:** Неправильные настройки БД или нет доступа

**Решение:**
- Проверьте переменные окружения БД
- Убедитесь, что база данных доступна из контейнера
- Проверьте firewall/security groups
